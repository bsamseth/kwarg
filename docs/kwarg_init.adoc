== NAME

kwarg_init, kwarg_init_from_opts - initialize an argument parser

== SYNOPSIS

[source,c]
----
// At least one .c file needs this before the include:
#define KWARG_INCLUDE_IMPLEMENTATION
#include <kwarg.h>

kwarg_parser_t kwarg_init(int argc, char *argv[], ...);

kwarg_parser_t kwarg_init_from_opts(
  int argc, const char *argv[static argc], kwarg_init_opts_t opts
);
----

== DESCRIPTION

Creates a parser from the command-line arguments recorded in *argc* and
*argv*. Must be called before any *kwarg_flag*(), *kwarg_str*(), or
*kwarg_finish*().

The macro form accepts optional named fields; the backend takes an explicit *kwarg_init_opts_t*.

*.no_help*:: *bool* (default false). Suppress the automatic *--help* / *-h* flag.
*.no_args_shows_help*:: *bool* (default false). Show help when *argc* == 1 (no user arguments).
*.tagline*:: *const char**. Short line printed at the top of help output.
*.description*:: *const char**. Description text printed after the tagline.
*.explicit_usage*:: *const char**. Custom usage line. The program name is prepended automatically.

== RETURN VALUE

Returns an *kwarg_parser_t* that must later be passed to *kwarg_finish*(3).

== EXAMPLES

[source,c]
----
kwarg_parser_t p = kwarg_init(argc, argv,
    .description = "Process data files.",
    .tagline = "mydata v1.0");
----

Explicit struct form:

[source,c]
----
kwarg_parser_t p = kwarg_init_from_opts(argc, argv,
  (kwarg_init_opts_t){
    .description = "Process data files.",
    .tagline = "mydata v1.0",
  }
};
----
