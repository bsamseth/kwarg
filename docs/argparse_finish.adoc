== NAME

argparse_finish, argparse_finish_from_opts - finalize argument parsing

== SYNOPSIS

[source,c]
----
#include <argparse.h>

argparse_result_t argparse_finish(argparse_parser_t *parser, ...);

argparse_result_t argparse_finish_from_opts(
  argparse_parser_t *parser, argparse_finish_opts_t opts
);
----

== DESCRIPTION

Finalizes argument parsing. This function:

1. Prints help to stdout if *--help* was given (and returns/exits).
2. Checks for unknown or unexpected arguments, writing diagnostics to stderr.
3. Validates that all *required* options and positional arguments were provided.
4. Prints usage to stderr on failure.
5. Frees internal memory.
6. Exits the program or returns a result, depending on the options.

*.no_exit_on_failure*:: Return ``ARGPARSE_PARSE_FAILED`` instead of calling *exit*.
*no_exit_on_help*:: Return ``ARGPARSE_HELP_INVOKED`` instead of calling *exit*(0).

== RETURN VALUE

``ARGPARSE_PARSE_OK`` (0):: Parsing succeeded.
``ARGPARSE_HELP_INVOKED`` (1):: Help was requested; programs should exit without taking further action.
``ARGPARSE_PARSE_FAILED`` (-1):: A parse error occurred; diagnostics have been written to stderr.

By default the function does not return on help or failure (it calls *exit*).
Set ``.no_exit_on_help`` or ``.no_exit_on_failure`` to get a return value instead.

== EXAMPLES

Typical usage:

[source,c]
----
argparse_finish(&p);
----

Non-exiting variant:

[source,c]
----
argparse_result_t r = argparse_finish(&p,
    .no_exit_on_failure = true,
    .no_exit_on_help = true);
switch (r) {
case ARGPARSE_PARSE_OK:      break;
case ARGPARSE_HELP_INVOKED:  exit(0);
case ARGPARSE_PARSE_FAILED:  exit(1);
}
----
